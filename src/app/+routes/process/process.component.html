Process

Prompt: You’re designing a mobile photo gallery interface. A designer on the team delivers mockups for a grid view and detailed photo view and asks you to help define the transition between the two states. If you have additional time, you can consider secondary actions, such as items in the detail view toolbar.

To better define the problem and the goal of this open exercise, I took the following process.

Defining Product Direction → UX/Interaction Design → Development

While my career focus is with UX/Interaction Design and Development, I wanted to take a more holistic approach by thinking of strategy and product direction. By thinking about strategy and product direction, we can define the audience and the business goals, which guides strategic design decisions. Furthermore, clearly defined design affords the opportunity to efficiently plan development.

Product Direction

Goal/vision: Create entertaining image editing features that could be integrated into any Google Chat App e.g. hangouts, allo, chat etc. (Note: Google Chat apps do not offer any entertaining image editing tools.)

Audience: Anyone who likes to communicate through photo and images.

Scenario/Persona: Jenny, 32 female— going on her first year of marriage. Living in San Francisco together, the couple are thinking of children but maybe they want to start with a dog. Jenny always takes pictures of dogs she sees waiting outside. Jenny likes to send the pictures to her husband, sometimes editing them on her phone, and always following up with, “I like this one!”

Product Requirement (User stories):

  • User can access photo reel (list of photos).

  • User can view a single selected photo.

Nice to haves (stretch goals if I have time):

  • User can view recently added photos.

  • User can add text to a photo.

  • User can add stickers to a photo.

  • User can draw on a photo.

  • User can view edited photo.

  • User can share edited photo.

  • Considerations for a spot in the UI to support future features.

  • Time for delivery: 2 weeks

UX/interaction design + technical planning

Once finished UI design, I started mapping out user flows.

In this step, I also defined interaction/motion patterns for transitions between individual screens

Transition from master view to detail view: I chose to scale the item directly and blend into the detail view. This follows material design’s motion principle, which guides user to the next view seamlessly.

Pagination of detail views I chose to allow swiping between images because heuristically it reduces number of interactions by 1. It’s also a common pattern among many established image gallery apps (like Google Photos). In parallel, I started annotating engineering details as well.

State Management: Redux/NGRX

Redux or Angular’s implementation @ngrx Is a robust library for managing state of an application. There are 3 principles to redux:

  • Single source of truth

  • State is read-only

  • Changes are made with pure functions

Inside the Project:

Actions: Actions describe state changes

Animations: Animation easing and blending constants.

Components: Stateless components that do not reference store.

Containers: Smart components that reference store.

Effects: Isolate side effects from components to fetch data.

Enums: Set of enums I use throughout the application to manage events.

Interfaces: Set of generic interfaces I use across the application

Reducers: Pure functions called reducers take the previous state and the next action to compute the new state

Services: Set of services used across the application.

Flow Control:

The data flows unidirectionally downwards, in order from:

  • Network: A 3rd party cms api : www.tipe.io

  • Angular router Service.

  • Into master view.

  • Data is rendered into items.

  • All Items are evaluated for position and cached.

  • On user click currently selected item is set in global store

  • Upon update to store a overlay is create and animations are triggered.

  • On finish of animation router triggers detail view.

  • Detail view overlaps transition detail-Item animation

  • The user interaction graphs out to close, swipe, bottom bar click

  • Each interaction fires and action to the global store

  • Each change in store will reactively trigger a ui state change.
